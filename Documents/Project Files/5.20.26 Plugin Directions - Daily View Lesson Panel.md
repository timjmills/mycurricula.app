# Plugin Directions — Daily View Lesson Panel

**Target:** Claude Code's frontend design plugin
**File scope:** The lesson detail panel (middle column) in the daily view of the MyCurricula webapp.
**Reference assets:**
- `Reference A` = the full lesson panel mockup (six numbered sections, Standards section expanded with resource grid + "More resources" list).
- `Reference B` = the "Resource quick access" minimized list (icon + title + subtitle + type pill row).
- `Current` = the screenshot of the live daily view shared on 2026-05-20.

Build the panel to match **Reference A exactly**, then add the minimize/thumbnail toggle described in §4. Apply every change below.

---

## 1. Header — adopt the weekly-view lesson-card style (REVISED 2026-05-20)

**Problem (Current):** The daily-view lesson header is a saturated dark-navy slab with small white text. The user wants it replaced with the **same visual style used by lesson cards in the weekly view**, with three repositioned content regions (subject big on the left, lesson name in the middle, time big on the right).

**Fix:** Re-skin the daily-view header to match the weekly-view lesson card, then arrange its content in three horizontal regions.

### 1.1 Background and shape

- Background: subject **light** tint — use design token **`var(--cl)`** for the active subject (Math resolves to `#e6efff` per `palette.jsx`).
- A 4px-wide solid left edge in **`var(--c)`** (subject base, Math = `#2b6cff`), like the weekly view card's accent stripe.
- 1px bottom border in `color-mix(in oklch, var(--c) 25%, transparent)`.
- No gradient, no shadow, no glow. Flat fill only.
- Bar height: **72px minimum** (taller than the previous slab to fit the three-region layout). Vertical padding: 14px. Horizontal padding: 20px.
- Text color across the bar: **`var(--cd)`** (subject deep, Math = `#1a3f99`). The whole header reads as one tone-on-tone unit, matching the weekly card.

### 1.2 Three-region content layout

Use a single flex row (`display: flex; align-items: center; gap: 16px`). Three children, in order:

**(A) LEFT — Subject, big.**
- The subject name ("MATH") in **uppercase**, **letter-spacing 0.6**, **weight 800**, **22px**, color `var(--cd)`.
- Optional: a tiny 2-letter abbreviation chip ("MA") in 11px, weight 700, color `var(--c)`, sitting to the left of the full name — matches the weekly card. Skip the chip if it crowds the row at narrow widths.
- A small empty checkbox glyph (the "Mark done" indicator from the weekly card) sits between the chip and the subject name, 16×16, `var(--ink-300)` border, 4px radius.
- `flex: 0 0 auto` — sized to content.

**(B) MIDDLE — Lesson name, centered.**
- The lesson title ("Equivalent fractions warm-up") in **weight 700, 19px**, color `var(--cd)`, single-line truncate with ellipsis on overflow.
- `flex: 1 1 auto; text-align: center; min-width: 0` so it occupies the center and centers within the remaining space.
- Optional second line beneath the title (smaller, 11.5px, `var(--cd)` at 70% opacity): the lesson's section count or sub-label. Omit by default; add only if the live data has it.

**(C) RIGHT — Time, big and right-justified.**
- The time range ("8:10–9:10") in **tabular numerals**, **weight 700, 26px**, color `var(--cd)`, `font-variant-numeric: tabular-nums`, `letter-spacing: -0.5`.
- `flex: 0 0 auto; text-align: right; margin-left: auto`.
- The kebab / overflow / expand icon buttons sit to the **right of the time**, vertically centered, in `var(--cd)` at 60% opacity, 16px glyphs in 28×28 hit targets, with 4px gap between buttons.

### 1.3 Required behavior

- The three regions must remain visually distinct: left content hugs the left edge (after the 4px accent stripe), the title is true-centered in the remaining width, and the time is hard-right-justified before the icon group.
- The header is a single row at desktop widths. Below 720px panel width, allow the subject and time to retain their sizes while the title truncates first; only at <540px should the title wrap or the time shrink to 20px.
- The 4px subject-base accent stripe runs the full height of the header on the left edge — flush, no padding inside it.

**Anti-goals:**
- No dark-navy slab; no white-on-dark text.
- No gradient or shadow on the header.
- Do not put the time on the same column as the subject or stack it under the subject — it must be right-justified on its own.
- Do not bold every word; the three regions are styled distinctly: subject = uppercase tracked, title = title-case bold, time = bold tabular numerals.

### 1.4 Visual reference

```
┌────────────────────────────────────────────────────────────────────────┐
│▌ MA ☐ MATH      Equivalent fractions warm-up        8:10–9:10  ⋮ ⛶  │
└────────────────────────────────────────────────────────────────────────┘
  ↑       ↑                  ↑                            ↑       ↑
 4px    left region      middle region (centered)     right region  icons
 accent (big subject)    (lesson title)              (big tabular time)
```

Light blue fill (`var(--cl)`), deep blue text (`var(--cd)`), no gradient.

## 2. Left-justify the lesson body

**Problem (Current):** The lesson title ("Equivalent fractions warm-up"), I-CAN pill, "Mark done"/"Add status" controls, and all six section blocks are horizontally centered inside the middle pane, with empty margins on both sides.

**Fix:** Left-align everything in the lesson body.

- The lesson body container should use `max-width: 760px` and **`margin: 0` on the left, `margin-right: auto`** — i.e., flush-left to the pane, not `margin: 0 auto`.
- Inner left padding off the pane edge: **32px**.
- Right padding: minimum 24px before the (existing) right rail (Resources panel, To-do List, Day Shoutbox).
- All section headers, paragraph copy, and the "Mark done"/"Add status"/"Lesson notes" row align to the same 32px left gutter.
- The "Expand all / Collapse all" link sits at this same left edge, above Section 1.

**Implementation hint:** the centered behavior is almost certainly coming from a `margin: 0 auto` or `justify-content: center` on a `.lesson-body` / `.lesson-content` container. Change to `margin-inline-start: 0; margin-inline-end: auto` (or remove the centering rule altogether) and let the parent flexbox align children to the start.

## 3. Section layout — match Reference A exactly

The body has six sections, in this order, each rendered identically:

| # | Title | Subtitle (helper) |
|---|---|---|
| 1 | Standards | Interpret a fraction as division of the numerator by the denominator (a/b = a ÷ b). |
| 2 | Focus Lesson — I Do | Model the concept and solve examples together. |
| 3 | Guided Instruction — We Do | Practice together with teacher support. |
| 4 | Collaborative Practice — You Do Together | Work in pairs or groups to apply the skill. |
| 5 | Independent Practice — You Do Alone | Students practice independently. |
| 6 | Debrief | Reflect on learning and key takeaways. |

### 3.1 Numbered badge (NEW component)

Each section begins with a **colored numbered circle** on the far left of the row. Per Reference A:

- Diameter: 26px circle, white-bold sans-serif numeral 13px center-aligned.
- A short vertical connector line (1.5px wide, same color as the circle, 50% opacity) runs from the bottom of the circle to the top of the next section's circle, creating a continuous timeline along the left edge.
- Color per section (in order, top to bottom):
  1. **Teal** — `#14b8a6`
  2. **Orange** — `#f97316`
  3. **Purple** — `#a855f7`
  4. **Amber/Yellow** — `#eab308`
  5. **Blue** — `#3b82f6`
  6. **Pink** — `#ec4899`
- The circle has a soft outer glow (`box-shadow: 0 0 0 4px color-mix(in oklch, <color> 18%, transparent)`).

### 3.2 Section title row

To the **right** of the numbered badge, on the same baseline:

- **Plain text title** at 15px, weight 600, color `var(--ink-900)`. **NO background highlight, NO colored fill behind the text.** This is the "highlighting style" the user wants reverted — current production fills the title with a blue gradient bar; remove that completely.
- A right-side caret chevron (`v`) for expand/collapse, in `var(--ink-400)`, 12px.
- Optional spacer to push the chevron to the far right.

### 3.3 Section helper line

Directly under the title, indented to match the title's left edge (so it lines up with the title, not the badge):

- Plain text, 12.5px, color `var(--ink-500)`, line-height 1.4.
- Content = the "Subtitle (helper)" column from the table above.

### 3.4 Section body (when expanded)

- A single multi-line text input: placeholder `"Write lesson plan for this section…"`.
- Style: full-width inside the section, 1px solid `var(--ink-150)`, `border-radius: 6px`, 10px padding, min-height 56px, font 13px, color `var(--ink-900)`, placeholder `var(--ink-400)`.
- Section 1 ("Standards") shows the resource panel docked to the **right** of this input (see §4). Sections 2–6 show resources stacked **below** the input when expanded, OR as the minimized row (see §4.3) when toggled.

### 3.5 Sections 2–6 are collapsed by default in Reference A

Render them as just badge + title + chevron + helper line. No body, no resources visible. The chevron points right (`>`) when collapsed.

## 4. Resources — per-section, with expanded ↔ minimized toggle (NEW)

Each section has its own resources area. Each one independently toggles between two states.

### 4.1 The toggle control

A small **icon button** lives at the top-right of the resources area:
- Two icons swap depending on state.
- **Expanded → minimize icon**: a "collapse" glyph (square with two arrows pointing inward), `var(--ink-500)`, 14px, in a 24×24 hit target.
- **Minimized → expand icon**: an "expand" glyph (square with two arrows pointing outward), same styling.
- Tooltip: "Minimize resources" / "Expand resources".
- Persist state per (lesson_id, section_id) in component state (and in localStorage if the app already does so for other UI prefs).

### 4.2 Expanded state — matches Reference A's Section 1

The expanded resource panel is a card docked to the right of the section body (in Section 1 it sits next to the Standards textarea; in sections 2–6 it sits below the textarea unless the section content is wide enough to put it inline — default to "below" for 2–6).

Structure (top to bottom inside the card):

1. **Header row**: label "Resources" (11px, `var(--ink-500)`, uppercase, letter-spacing 0.6). Right side: the minimize toggle (§4.1).
2. **Primary grid** — 2×2 of resource thumbnails (cards):
   - Card size ~92×92px, `border-radius: 10px`, padding 12px.
   - Each card has a colored soft-tint background and a small monochrome icon centered, with the title beneath, 11.5px, weight 600, color `var(--ink-900)`, line-height 1.25, truncate at 2 lines.
   - Card 1: "Fraction Basics" — soft purple fill `#f3e8ff`, play-triangle icon.
   - Card 2: "What is a Fraction?" — soft teal fill `#ccfbf1`, paperclip icon.
   - Card 3: "Fractions Overview" — soft green fill `#dcfce7`, document icon.
   - Card 4: "Khan Academy" — soft peach fill `#fed7aa`, paperclip icon.
3. **"+ Add resource" button**: a centered text+icon link beneath the grid, 12.5px, color `var(--cd)` (subject deep), with an underline-on-hover. No border.
4. **More resources** sub-panel — a stacked list of additional resources (rows):
   - Section header: "More resources" 11px, `var(--ink-500)`, uppercase, letter-spacing 0.6.
   - Rows: icon (16px, neutral gray) + title (12.5px, weight 500, `var(--ink-900)`) + right-aligned type pill (see pill spec in §4.4).
   - 3 rows in Reference A: "Fraction Wall Poster" (PDF), "Anchor Chart Template" (DOCX), "Fraction Examples Sheet" (PDF).
   - Below the list: "Show more v" link, 11.5px, `var(--ink-500)`.

### 4.3 Minimized state — matches Reference B's "Resource quick access"

When the user clicks minimize, the entire expanded card (primary grid + More resources sub-panel) collapses to a **single compact card**:

Structure:

1. **Header row**: small link/paperclip glyph + label "Resource quick access" (12px, weight 600, color `var(--ink-700)`). Right side: the expand toggle (§4.1).
2. **List rows** (one per resource, vertically stacked, separated by a 1px `var(--ink-100)` line; no zebra striping):
   - Left: a 22×22 rounded square (`border-radius: 6px`) containing a monochrome icon. Fill color is a light tint per type:
     - Video → light pink `#fce7f3`, play-triangle icon, pink stroke `#db2777`.
     - DOCX → light blue `#dbeafe`, document icon, blue stroke `#2563eb`.
     - PDF → light orange `#ffedd5`, document icon, orange stroke `#ea580c`.
     - Link → light gray `#f3f4f6`, link-chain icon.
   - Middle (flex: 1): two-line stack
     - Line 1: title, 12.5px, weight 500, `var(--ink-900)`, single-line ellipsis.
     - Line 2: subtitle (the source/URL preview, e.g. "youtube.com/watch?v=…" or "drive.google.com/…"), 10.5px, `var(--ink-500)`, single-line ellipsis.
   - Right: **type pill** (see §4.4).
3. Reference B's three rows (drive the seed data):
   - "Fraction Basics Video" / `youtube.com/watch?v=…` / Video pill.
   - "Fraction Anchor Chart" / `drive.google.com/…` / DOCX pill.
   - "Fraction Examples Worksheet" / `drive.google.com/…` / PDF pill.
4. **Footer**: centered "+ Add quick resource" link, 12.5px, color `var(--cd)`, padded 10px above/below, with a 1px dashed top border `var(--ink-150)`.

Row vertical padding: 8px. Card outer padding: 14px. Card `border-radius: 12px`, background `var(--paper)`, border `1px solid var(--ink-150)`.

### 4.4 Type pill spec

A small rounded pill on the right of each minimized row (and each "More resources" row in §4.2):

- Padding: 2px 8px; `border-radius: 999px`; font 10px, weight 600, uppercase, letter-spacing 0.4.
- Color scheme (background / text):
  - **VIDEO** — `#fce7f3` / `#9d174d`
  - **DOCX** — `#dbeafe` / `#1e40af`
  - **PDF** — `#fee2e2` / `#991b1b`
  - **LINK** — `#f3f4f6` / `#374151`
  - **SLIDES** — `#fef3c7` / `#92400e`
  - **IMAGE** — `#dcfce7` / `#166534`

## 6. Acceptance criteria

The change is done when:

1. The header bar is a single flat row in `var(--cl)` with `var(--cd)` text and a 4px `var(--c)` left accent stripe (weekly-card style). Three regions: **left** = big uppercase subject ("MATH", 22px weight 800) with mark-done checkbox glyph; **middle** = centered lesson title (19px weight 700); **right** = big right-justified time ("8:10–9:10", 26px weight 700, tabular numerals) followed by kebab + expand icons. No dark-navy fill, no gradient, no shadow.
2. No section title has a colored highlight bar behind it — section titles are plain dark text only.
3. Lesson body content (title, I-CAN, action row, all sections, Expand/Collapse links, Add section button) is left-aligned to a 32px gutter from the pane's left edge; nothing is horizontally centered except the Add Section / Edit flow controls at the bottom.
4. Each section row leads with a 26px numbered circle in its assigned color (teal / orange / purple / amber / blue / pink) with a vertical connector line linking the six circles.
5. Section 1 is expanded by default and shows the expanded Resources card with the 2×2 thumbnail grid and "More resources" sub-list.
6. Sections 2–6 are collapsed by default.
7. Every section's Resources area has a working minimize/expand icon button in the top right.
8. Clicking minimize replaces the expanded card with the compact "Resource quick access" list (icon square + title + URL-preview subtitle + type pill).
9. Clicking expand on a minimized card restores the full expanded card with the user's prior data intact.
10. State is persisted per section while the user navigates between lessons in the same session.
