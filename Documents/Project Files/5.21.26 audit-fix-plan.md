## Working rules

- The build must be green (`npx tsc --noEmit`, `npm run lint`, `npm run format:check`)
  at the end of every wave before its commit.
- Tokens only — no hard-coded hex/px font sizes in components (`app/tokens.css` is the
  source of truth). Tailwind for layout/spacing only.
- Never hard-code the weekday set or the daily schedule — derive from config.
- Do not break the forking model; do not add confirm dialogs for Master mode.
- Waves run sequentially; agents within a wave own disjoint files.

## Wave 1 — Critical bugs (geometry + data binding)

- [ ] **BUG-001 / DAILY-CLOCK-001 / MED-1** — Planner + right rail become a CSS grid
  (`1fr var(--rail-w)`); rail no longer overlaps the grid/banner. Hide-rail toggle
  collapses `--rail-w` to a stub; below 1280px the rail hides and is reachable as a
  sheet. *Accept:* rail vs grid `getBoundingClientRect()` never intersect at
  1024–1920px; `/daily` banner shows the full `8:10–9:10` time with no clipping.
- [ ] **BUG-005** — Right-rail Resources panel scopes to the selected lesson on
  `/weekly` (uses `selectedLessonId` from `app-state`). *Accept:* selecting a card
  changes the rail title to that lesson + count to its resource count ≤100ms;
  Esc / canvas click reverts to week aggregate.
- [ ] **BUG-006** — One resource source per lesson event: the rail, the weekly card,
  and the daily lesson all read the same resources for a lesson. *Accept:* a resource
  added on any surface appears on the others for the same lesson.
- [ ] **BUG-002** — Weekly card title band restructured to two rows so subject/time/
  drag/counter/menu/MODIFIED never overlap. *Accept:* no two interactive band
  elements intersect in any state combo {clean, modified, swap, counter, checkbox}.
- [ ] **CARD-TITLE-002** — Inline title editor has `maxLength` + 2-line clamp.
  *Accept:* pasting 200 chars leaves the card at normal height; rows below do not shift.
- [ ] **BUG-003** — Sticky subject row-head cells get an opaque background token.
  *Accept:* horizontally scrolled grid keeps row labels fully opaque, no bleed-through.
- [ ] **BUG-004 / MED-6** — Subject-view STANDARDS filter derives from the active
  subject. *Accept:* Math→Reading swaps the STANDARDS pills within ≤100ms.
- [ ] **DAILY-STANDARDS-001** — `/daily` Standards section binds to the selected
  lesson's standards. *Accept:* switching lesson updates Standards within one render.

## Deferred (with rationale)

- **BIG-6 (Storybook per card state + CI a11y gate)** — Deferred. Storybook is a heavy
  new dependency; `CLAUDE.md §6` forbids new dependencies without clear need and the
  project has no CI harness yet. The card-state coverage intent is met instead by the
  two-row band restructure (BUG-002) being verified across every state combination by
  hand. Revisit when a CI pipeline exists.
- **PERSIST-002 (server persistence of mark-done)** — Out of scope: no backend yet;
  the app is a mock-data prototype (`CLAUDE.md §3`).
