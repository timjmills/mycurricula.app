### Documented deviations from the prototype (all deliberate)

- Status chips are click-to-cycle (prototype renders them static); labels use the design system's sentence-case vocabulary ("In progress", not "In Progress").
- Minutes are a structured `minutes` field, not parsed from title text.
- Closed planning tabs return via the "+ Add a tool" menu (production has no sidenav "Context" drop target).
- Tab/panel layout persists per-teacher in localStorage; server persistence is Phase 1B, same as the dock.
- Cross-phase resource drops append to the target list (store's move action has no index).
- The lesson-flow ··· toolbar was removed per the exact phaseHead anatomy — duplicate-section / show-on-website lose their UI (store actions remain).
- One literal `rgba()` (planning-tab active seat shadow) — no upward `--sh-*` token exists; the triple matches the tokens' shadow ink.
