## 3. Workspace / account model — the one real reconciliation

The handoffs specify a **simple** model: `profile.role ∈ {team, solo}`, a flat
`workspaces{ active, personal[], teams[] }`, one `team{ name, school, members[] }`, a flat
`curriculums{ active, list[] }`, and (7.21) a per-unit **share link** (a copy-link, not an
ACL). `config.jsx` is byte-identical across the whole stack.

But **richer product decisions were locked (7-17) and are partly shipped**: solo/team
**derived per workspace**, per-course sharing (personal = invisible to team; creator +
admin control), and **full multi-workspace + a v2 workspace-first wizard**. W12b-1 (the
course-sharing RPC seam) and W12c (workspace-first onboarding) already landed.

**Resolution:** the app-model intentionally **exceeds** the handoff's design truth here.
Treat the handoff `config.jsx` as the **visual/section template** for Setup, but wire it
to the **shipped richer model** — not the flat prototype model. This is the crux the
Settings/Config wave must get right (§5.1).

Remaining after W12a/b-1/c:
- **Full Setup ConfigPage** — build from `6.24 source/config.jsx` as the visual/section
  template (byte-stable across the whole handoff stack → low visual risk), reconciled with

  data/RLS, not visual).

  onboarding column lands (latent foot-gun until then).

### 5.2 The 7.21 Unified Unit/Lesson Workspace (Track B, B1–B5)
The flagship remaining surface. **A data-layer wave** — units are read-only in the seam
today, so this needs an **additive, nullable migration** (timestamp after
`20260717120000`): editable `units` fields (notes/big_idea/essential_questions/vocab/
standards/…), lesson `taught_at` + rich fields **added to all three fork tables**
(`master_core_lesson_events`, `personal_core_lesson_event_copies`,
`personal_authored_lessons`), and a new `planner_settings` table.
- **B1** shell + Unit Plan · **B2** Lessons editor · **B3** Assessments + Insights +
  drawer · **B4** Shared Composer (absorb `components/daily/ResourceComposer`) · **B5**
  pop-in overlay + retire `UnitDrawer`/focus-lesson/`SubjectView`.
- B0 is done; a ready ordered B1 sub-plan (B1.0–B1.8) lives in `agent_shared_log.md`.
- Reuse (don't rewrite): `ExplorerShell`, `UnitExplorer`, `PlanPage` + its tabs,
  `lib/year-v2-data.ts`, `lib/year-unit-aggregate.ts`, the hub-v2 browse/search layer.
- **Porting hazards to avoid** (from B0): model `unit.framework` + the resolution chain
  (never leave `u.fw` unassigned); derive "remaining" from the configured school week
  (Sun–Thu), not a hardcoded /5; write `taught_at` (don't rely on `status` inference);

  claims it reached the team); `Lesson.time` has **no DB column** (a time-only write
