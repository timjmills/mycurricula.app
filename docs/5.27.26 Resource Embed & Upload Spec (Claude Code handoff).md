# 5.27.26 Resource Embed & Upload — Research + Fix Spec

> **For:** Claude Code (WebStorm), which has the live `mycurricula.app` repo. This doc was written in Cowork from the project docs/design/deploy folder — the live `components/`, `app/`, `lib/` source was **not** accessible there, so Phase 0 below is a *verify-against-code* step, not a finished diagnosis.
> **Goal:** make real files, links, YouTube videos, slides, and PDFs actually embed/render, and let teachers attach files — as the foundation the **Teach view** (build starting next week) will sit on.
> **Stack (confirmed from deploy config):** Next.js + Supabase (auth + DB) + Cloudflare Worker (wrangler). Design intent for resources is a **Padlet-style board** (`Claude Design/.../artboards-resources.jsx`).

---

## Hard requirements (Tim, 2026-05-27)

| Resource type | Per-lesson limit | Size cap each |
|---|---|---|
| Files (PDF, PPTX, DOCX, etc.) | up to **10** | **≤ 25 MB** |
| Photos / images | up to **10** | **≤ 5 MB** |
| Links (external URLs) | **unlimited** | n/a |

**Links must render three ways**, teacher's choice per link:
1. **Literal link** — the raw URL shown as plain visible text (e.g. `https://…`).
2. **Hyperlinked text** — custom anchor text linking to the URL (e.g. "Prime factorisation video").
3. **Thumbnail preview** — a visual embed/preview card on the resource board.

---

## Phase 0 — Confirm root causes against the live code

The likely causes below are inferred from the design artboard, the two UI/UX audits, and the stack. Verify each before building; tick or correct.

1. **No render layer turns a resource into an embed.** In the design, cards carry a `kind` (`pdf`/`pptx`/`youtube`) plus a `thumb` *key* (a placeholder glyph) — never a real `<iframe>`/`<img>`/PDF viewer. Audits confirm live rows are "non-interactive" with "abstract glyphs." → Check `components/subject/ResourcesSort.tsx`, `components/daily/ResourceComposer.tsx`, `components/lesson-flow/section-resources.tsx`, `components/daily/LessonDetail.tsx`. **Q: is there any component that renders an embed, or only labels/badges?**
2. **No file storage/upload pipeline.** The Supabase deploy SQL defines only auth-bypass + audit-log tables — **no Storage bucket**. The Upload control and the Resource Composer "Search the resource board — coming soon" tile are stubs. → **Q: confirm there is no `resources`/`lesson-files` bucket and no upload API route.**
3. **Pasted links aren't normalized for embedding.** YouTube `watch?v=`, Google Slides `/edit`, Drive `/view` each need rewriting to `/embed` or `/preview`. → **Q: search `lib/` for any URL-transform/oEmbed helper (expected: none).**
4. **CSP / frame headers block embeds.** The classic failure. → **Check `next.config.js`/`next.config.ts` `headers()`, any `middleware.ts` CSP, and the Cloudflare Worker response headers for `Content-Security-Policy` (`frame-src`/`img-src`/`media-src`) and `X-Frame-Options`. Open a resource board and look for CSP violations in the browser console.**
5. **PDF served as download, not inline.** Supabase Storage defaults to attachment disposition. → **Q: how are/will files be served — public URL, signed URL, or proxied? Determine where to set `Content-Disposition: inline`.**
6. **Data model.** → **Q: how are resources stored today — a Supabase table, mock fixtures, or localStorage? Is there a `resources` table with a `lessonId` FK?** This determines how much model work is net-new.

## Fix architecture

### A. Data model

A `resources` table (Supabase), one row per attachment, FK to lesson:

```
id            uuid pk
lesson_id     uuid fk -> lessons(id)
team_id       uuid   -- for RLS scoping
kind          text   -- link | youtube | vimeo | gslides | gdocs | gsheets | gdrive | pdf | image | video | audio | file
display_mode  text   -- 'literal' | 'hyperlink' | 'thumbnail'   (links only; default 'thumbnail')
link_text     text   -- anchor text when display_mode='hyperlink'
url           text   -- external URL (links/embeds) OR null for uploads
storage_path  text   -- Supabase Storage path for uploads OR null for links
title         text
caption       text
mime_type     text
size_bytes    bigint
width         int     -- images/video, nullable
height        int
position      int     -- ordering on the board
created_by    uuid
created_at    timestamptz default now()
```

Enforce counts **server-side** (DB trigger or API guard): ≤10 rows where `kind='image'` per `lesson_id`; ≤10 rows where `kind IN (file/pdf/pptx/doc…)` per `lesson_id`; links unlimited.

### B. URL parser / normalizer — `lib/resource-embed.ts`

Pure function: `parseResourceUrl(raw: string) => { kind, embedUrl, thumbnailUrl, displayName, canEmbed }`.

- **YouTube** `watch?v=ID` / `youtu.be/ID` → `https://www.youtube-nocookie.com/embed/ID`; thumbnail `https://img.youtube.com/vi/ID/hqdefault.jpg`.
- **Vimeo** → `https://player.vimeo.com/video/ID`.
- **Google Slides** `/presentation/d/ID/edit` → `/embed?start=false&loop=false`.
- **Google Docs / Sheets** `/d/ID/edit` → `/preview` (Sheets: `/preview` or `/pubhtml`).
- **Google Drive file** `/file/d/ID/view` → `/file/d/ID/preview`.
- **PDF URL** (ends `.pdf` or known viewer) → inline viewer.
- **Generic link** → server route fetches OpenGraph metadata (`og:title`, `og:image`) for the thumbnail; fall back to favicon + domain. Mark `canEmbed=false` for sites that send `X-Frame-Options: DENY`.
- **Security:** accept only `http`/`https`; reject `javascript:`/`data:` schemes; never inject raw HTML.

Add a unit test file covering each provider + a malformed URL.

### C. `<ResourceEmbed>` renderer

Switch on `kind`:

- `youtube|vimeo|gslides|gdocs|gsheets|gdrive` → sandboxed `<iframe loading="lazy" allow="…">` inside a responsive aspect-ratio wrapper.
- `image` → `<img loading="lazy">` with intrinsic width/height.
- `video` / `audio` → native `<video controls>` / `<audio controls>`.
- `pdf` → `<iframe>` to the inline URL (or PDF.js if inline serving is blocked).
- `link` + `display_mode='thumbnail'` → preview card (OG image/title/domain).
- `link` + `display_mode='hyperlink'` → anchor element using `link_text` as the visible text (see G).
- `link` + `display_mode='literal'` → the raw URL as visible, clickable plain text.
- **Fallback** (link rot / non-embeddable) → a graceful card with title + "Open in new tab," never a broken iframe.

### D. Upload pipeline (Supabase Storage)

- Buckets: `lesson-images` (5MB cap) and `lesson-files` (25MB cap) — or one `lesson-resources` bucket with per-mime validation. Private, with RLS.
- **RLS:** read = same `team_id`; write = team member who owns the lesson scope (tie to the Personal/Team model). Mirror whatever lesson-table RLS exists.
- **Limits enforced server-side** in the upload API route: reject >25MB files / >5MB images / the 11th of either type per lesson; validate mime against an allowlist (pdf, pptx, docx, xlsx, png, jpg, webp, gif, mp4, mp3…).
- **Inline serving:** set `contentType` + `cacheControl` on upload; for PDFs/images generate signed URLs with `download: false` (or `response-content-disposition=inline`) so they display rather than download.
- **Images:** consider `sharp` to downscale + generate a thumbnail on upload (keeps the 5MB cap comfortable and the board fast).
- **UI:** replace the stub Upload tile / Composer search placeholder with drag-and-drop + file picker, upload progress, per-file error states, and the count/size limits shown inline.

### E. CSP / headers

Add or extend the CSP (in `next.config` `headers()` and verify the Cloudflare Worker doesn't override it):

```
frame-src  'self' https://www.youtube-nocookie.com https://www.youtube.com https://player.vimeo.com https://docs.google.com https://drive.google.com;
img-src    'self' data: blob: https://*.supabase.co https://img.youtube.com https://*.googleusercontent.com https://*.ggpht.com;
media-src  'self' blob: https://*.supabase.co;
```

Add the actual Supabase project domain. Confirm the app isn't sending `X-Frame-Options: DENY` in a way that interferes, and that the Worker passes these headers through.

### F. Edge cases & quality

- Link-rot 404 fallback (flagged as a gap in the UI/UX audit).
- Non-embeddable providers → link card, not a blank iframe.
- A11y: every embed/card has a title + keyboard focus; iframes have `title=`.
- Perf: lazy-load all iframes/images (the board can hold many resources; Year/Catch-up already render large DOMs).

### G. Teach-view hook (forward-looking)

The Teach view's writing area needs a **link-insertion** that can drop a resource in any of the three presentations — **literal URL**, **hyperlinked text** (custom anchor text), or a pinned **thumbnail** card. Build `display_mode` + `link_text` into the model now (§A) so the Teach view editor can flip a link between all three without a schema change later.

## Suggested build order

1. **Backend foundation** — `resources` table + buckets + RLS + upload API with limit/mime enforcement.
2. **Normalizer** — `lib/resource-embed.ts` + unit tests.
3. **Renderer** — `<ResourceEmbed>`; replace placeholder cards in the resource board.
4. **Composer UI** — drag-drop upload + paste-a-link with the thumbnail/inline-link toggle.
5. **CSP** — allowlist; verify embeds render with no console violations at 360/768/1280.
6. **Polish** — fallbacks, a11y, lazy-load, image thumbnails.

## Verification checklist

- Paste a YouTube link, a Google Slides link, a Drive PDF link, and a plain website link → each shows an embed or a link card (not raw text).
- Toggle a link between literal URL, hyperlinked text, and thumbnail → all three render correctly.
- Upload a 25MB PDF (accepted), 26MB (rejected with clear copy), an 11th file (rejected), a 5MB image (accepted), a 6MB image (rejected).
- A PDF opens **inline**, not as a download.
- No CSP violations in the console; embeds visible across breakpoints.
- RLS: a teammate can view the lesson's resources; a non-team user cannot.
- `npm run lint && npm run build` clean.
