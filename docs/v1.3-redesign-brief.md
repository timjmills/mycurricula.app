## 0. Migration strategy (decided with the brand owner)

1. **Remap values, keep names.** We do **not** mass-rename the app's existing tokens
   (`--ink-900‚Ä¶--ink-50`, `--t-*`, `--r-*`, `--math` ‚Ä¶). We change their **values** to
   the v1.3 palette and **add** v1.3's new tokens (`--brand-*`, `--honey-*`, `--subj-*`,
   `--canvas`, gradients, soft shadows, generous radii, Poppins/DM Sans). Existing
   surfaces re-theme automatically; new work uses the new names.
2. **Full overhaul** ‚Äî every surface (shell, Weekly, Daily, Year, Subject, Catch-up,
   Schedule, Settings, Onboarding, Login, marketing landing, Teach board + widgets).
3. **8 active subjects, 15 available.** Keep the locked 8 subject IDs
   (`math, reading, writing, grammar, spelling, ufli, explorers, sel`). Re-point their
   colors onto the v1.3 15-slot muted scale; expose all 15 slots as palette options.
   **Authoritative subject ‚Üí slot map** (from the kit's `data.js`):
   `math‚Üí1, reading‚Üí10, writing‚Üí2, grammar‚Üí7, spelling‚Üí5, ufli‚Üí3, explorers‚Üí13, sel‚Üí9`.
4. **Type sizes stay; families change.** Keep the app's existing `--t-*` size scale
   (the planner is intentionally dense). Change the *families*: headings ‚Üí Poppins /
   DM Sans; body/UI ‚Üí Plus Jakarta Sans. Do **not** bump base font-size.
5. **Keep the contracts:** `data-style` (quiet/calm/vivid), `data-palette`
   (normal/highlight), `.cp-subj.<id>`, the forking model, the tooltip system, and the
   `Button`/`Badge`/`Card`/etc. primitive APIs. Reskin them; don't break their props.

## 1. The look in one paragraph

Dashboard clarity, landing-page warmth. White surfaces on a **warm cream canvas
`#FCFAF6`** lifted by a barely-there fixed atmospheric mesh (honey top-left, violet
top-right, green bottom-right). **Indigo `#3B6CF6`** does the work (buttons, links,
focus, active nav, in-progress); **honey `#F4B740`** does the welcome (marketing CTA,
eyebrows, needs-review). Ink is a near-black violet `#1C1B2E`, never pure black.
**Color is navigation and it cascades**: a subject hue flows to its units, weeks and
lessons via `--c` (bright accent ‚Üí outline/tile/dot) + `--ct` (tint fill); chip **text
stays dark ink**. Status (green done / blue in-progress / grey not-started / honey
needs-review) is a separate, more-saturated layer. Generous radii (cards 18, panels
24, **pills 999** for buttons/badges/search), hairline warm borders, soft cool
shadows, color glows **only** under primary/honey buttons. Sentence case, no emoji.

## 3. Component recipes (mirror `ui_kit.css` / `marketing.css` exactly)

Use tokens only ‚Äî **zero hex** in component CSS. Match these to the screenshots.

- **Button** (`--r-pill`, weight 700): `primary` = `--brand-500` fill, white text,
  `--sh-brand` glow, hover `--brand-600`. `honey` = `--grad-honey`, text `#3A2A05`,
  `--sh-honey`. `secondary` = `--surface`, `--border`, `--ink-soft`, `--sh-xs`, hover
  border `--brand-300` + text `--brand-600`. `ghost` = transparent, hover `--hairline`.
  `destructive` keeps catchup/danger. Press settles `translateY(1px)`.
- **Eyebrow / label:** 11px 700‚Äì800, `+0.09em`, uppercase; honey variant =
  `--honey-600` on `--honey-50` with `--honey-100` border, pill. The one uppercase style.
- **Card:** `--surface`, `1px var(--border)`, `--r-lg` (18), `--sh-sm`, 24px pad;
  hover lifts `translateY(-2/-3px)` + `--sh-md`. No colored left-border by default.
- **Lesson card** (cascade): `--surface`, `border-left:4px solid var(--c)`,
  `--c-border` border, `--r-md`, `--sh-xs`; subject dot `--c` + name + time meta;
  title `--font-sans` 700 dark ink; standard pill `--ct`/`--ck` mono; status ring on
  the right; `.modified` ‚Üí dashed left border + honey "Edited" badge; `vivid` style
  fills with `--ct`.
- **Cascade chips** (`uchip`/`wchip`/`lchip`): `--ct` fill, `1.5px solid var(--c)`
  outline, soft drop shadow, **dark ink text**; unit = leading solid `--c` tile (white
  glyph/numeral), week = soft neutral tile (`--c` glyph), lesson = `--c` dot;
  `.active` ‚Üí 2px outline.
- **Badge:** pill + leading dot; `b-done/b-prog/b-idle/b-warn` use status tints.
- **Status ring:** done = filled `--done` + white check; prog = `--progress` ring with
  center fill; idle = hollow `--idle` ring.
- **Segmented control (`seg`):** `--hairline` track, `--border`, pill; active button =
  `--surface` + `--brand-600` + `--sh-xs`. (Maps to our `ToggleGroup`.)
- **Toggle:** 42√ó24 pill, off `--idle`, on `--brand-500`, white knob.
- **Panel:** `--surface`, `--border`, `--r-lg/-xl`, `--sh-sm`.
- **Top bar:** translucent cream `rgba(252,250,246,.82)` + `blur(12px)`, `--border`
  bottom; brand glyph in honey-gradient tile w/ `--sh-honey`; wordmark **DM Sans 800**
  `mycurricula` + `.app` in `--honey-600`; view tabs pill, active = `--brand-50` /
  `--brand-700`; Personal/Team `seg`.
- **Icon rail:** translucent white, 40px items `--r-md`, active = `--brand-500` fill +
  `--sh-brand`.
- **Team-curriculum banner:** soft pink gradient `#FDECEF‚Üí#FBE3E8`, border `#F4CBD6`,
  text `#9A2E42`. (Maps to our master-banner; keep the always-on tooltip + flash rules.)
- **Avatar:** `--grad-dawn` circle, white initials, 2px white ring.
- **Headings:** apply `--font-display` (Poppins) to display/H1 (page titles), and
  `--font-display-sm` (DM Sans) to H2/H3/card titles. Body stays `--font-sans`.

## 5. Rules (from README ‚Äî non-negotiable)

- Sentence case everywhere; only the eyebrow/label is uppercase. Status vocabulary is
  fixed: **Completed ¬∑ In progress ¬∑ Not started ¬∑ Needs review**.
- Gradients are atmosphere only (hero, onboarding, empty states, CTA) ‚Äî never on
  ordinary cards/panels. Color glows only under primary/honey buttons.
- Respect `prefers-reduced-motion`; keep the master-banner flash + urgent-pulse rules.
- Keep WCAG AA contrast, full keyboard nav, ‚â•44px touch targets, the dismissible
  tooltip system, and the responsive contract (phone 360‚Äì480 / tablet 600‚Äì900 /
  desktop 1024‚Äì1920; no document-level horizontal scroll at any tier).
- Tailwind = layout/spacing utilities only. All color/type/radius/shadow via tokens.
