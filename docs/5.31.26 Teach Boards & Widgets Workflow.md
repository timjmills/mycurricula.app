# Teach Boards & Widgets — Dynamic Workflow (agent-team runbook)

> Operational companion to `5.31.26 Teach Boards & Widgets Ultraplan.md`.
> This is the execution sequence: who builds what, in what order, with which
> gates and stop conditions. Dated 2026-05-31. Branch: `claude/ecstatic-bohr-1ovo3`.

## Roles

- **Lead (me).** Owns the shared contract (tokens, model, theme module, repo
  seam), all central wiring, and every verification gate. Builds Wave A and
  Wave C serially. Never lets a parallel agent touch a shared-contract file.
- **Build agents (Wave B).** Each owns a disjoint directory. They consume the
  frozen contract from Wave A; they never edit `lib/types.ts`, `app/tokens.css`,
  `lib/teach/queries.ts`, `lib/teach/mock-source.ts`, or the theme module.

## Gate definitions (a wave does not advance until its gate passes)

- **G-types:** `npx tsc --noEmit` → 0 errors.
- **G-lint:** `npm run lint` clean + `npx prettier --check` on touched files.
- **G-build:** `npx next build` green (run at end of Wave C/D only — expensive).
- **G-tokens:** grep finds no `#hex` and no `font-size:\s*\d+px` in touched
  components (`rg -n '#[0-9a-fA-F]{3,6}'` / `rg -n 'font-size:\s*[0-9]+px'`).
- **G-privacy:** grep finds no student-name fields in fixtures/config/state
  (`rg -ni 'firstName|lastName|roster|studentName'` → only comments/avatars).
- **G-responsive:** `node scripts/probe-uxa.mjs` (or DevTools emulation notes)
  at 360 / 768 / 1280 — no doc-level horizontal scroll; ≥44px targets.
- **G-codex:** `codex exec --sandbox read-only "$REVIEW_PROMPT"` →
  `NO BLOCKING ISSUES`. **Owner-run** (401 in this container). If it cannot run,
  report the blocker; never weaken the sandbox.

## Sequence

### Wave A.0 — Schedule/lesson-model survey (LEAD, read-only) — BLOCKS A

Map the real slot/lesson/day/week/subject entities + the planner selectors that
answer "what's at day D / week W / subject S / slot T". Output feeds the frozen
`RepeatRule` shape so Repeat creates REAL links (owner decision: real links now).
Gate: survey report in hand + repeat shape confirmed against true ids.

### Wave A — Foundation (LEAD, serial) — BLOCKS everything

1. Tokens: add the 5.31 visual-system section to `app/tokens.css` (fonts, 6
   families, neutrals, radii, shadows). → **G-tokens, G-types**
2. Model: extend `lib/types.ts` — `BoardPage`, `CanvasWidget` (x/y/w + ov),
   `ThemeOverride`, `RepeatSchedule`; add `pages`/`repeat`/`boardTheme` to
   `Board`; keep existing optional fields. Provide grid→x/y mapper.
3. Theme module: `lib/teach/widget-theme.ts` — `effective`, `clean`,
   `themeVars`, `BG_OPTS/ACCENT_OPTS/TEXT_OPTS/FONT_OPTS`, `WIDGET_DEFS`
   defaults. Pure, no React.
4. Repo: extend `TeachDataSource` + mock — page-aware reads, `setBoardRepeat`,
   canvas widget upsert/move/resize, keep library/cap methods. Update fixtures
   (tone-mapped tags, repeat samples, pages) — structure only.
5. **GATE A:** G-types + G-lint + G-privacy. Commit “Wave A foundation”.
   Only then spawn Wave B.

### Wave B — Build (PARALLEL agents, disjoint dirs)

Spawn together; each returns its own G-types/G-lint result.

- **B1 Widgets** — `components/teach/widgets/**`
  Shared `WidgetCard`/`WidgetHeader`/chrome + line-icon set first; then the ~40
  widgets (named + restyled generics), catalog rebinned to 6 categories,
  interactivity per spec, privacy avatars. May split B1a (Lesson/Management/
  Assessment) + B1b (Language/Well-Being/Utilities + restyled generics).
- **B2 Editor** — `components/teach/board/editor/**`
  Free-form canvas, drag/resize/select tools, appearance panel (board vs
  per-widget), pages bar, resource DnD modal, toolbar.
- **B3 Fullscreen** — `components/teach/board/fullscreen/**`
  Full-bleed bg presets, favorites bar, widget-library popup, markup panel
  (reuse annotation engine), page nav, chrome.
- **B4 Library** — `components/teach/library/**`
  Board Library refactor (Team/Personal, 50-meter, sidebar filters, cards with
  Repeat line + actions, Team Library, explainer) + Widget Library browser +
  Repeat-schedule UI.

**Collision rule:** if two agents need a shared helper, it belongs in Wave A —
escalate to lead, don't duplicate. Agents import widgets/theme from barrels,
never deep paths.

**GATE B (per agent):** G-types + G-lint + G-tokens on its own files. Lead
reviews each agent's diff before integration.

### Wave C — Wiring (LEAD, serial)

1. Mount Library → Editor → Fullscreen as one navigable flow in/around
   `TeachWorkspace`; deep links (`?board`/`?page`/`?present`).
2. Confirm nav order (Teach last — already true; no edit unless drift).
3. Remove/relabel dead controls per Wave-1 audit (H3/M4/M5, “Soon” honesty).
4. Reconcile reducer/state with pages + free-form (retire `BoardLayout`).
5. **GATE C:** G-types + G-lint + G-tokens + G-privacy.

### Wave D — Verify & package (LEAD)

1. G-build (`next build`).
2. G-responsive at 360/768/1280; a11y pass (focus traps, tab order, contrast).
3. Self-review every diff against Ultraplan §8.
4. **G-codex (owner-run).** Report result; fix Criticals/Highs; re-run.
5. Single squashed, well-described commit; push to the branch. **Do not open a
   PR or merge unless the owner asks.**

## Stop conditions (halt and ask, don't guess)

- A handoff detail is ambiguous AND changes data shape or destructive behavior.
- An agent needs to edit a shared-contract (Wave A) file → escalate to lead.
- A gate fails in a way that implies a spec contradiction (not just a bug).
- The 50-cap, Repeat semantics, or privacy rule would be violated to make
  something fit → stop, surface it.
