## 0. Decisions locked

1. **Build the full spatial model end-to-end** (backend + frontend) for the **v1
   collapse made real**: one Workspace can hold **multiple Notebooks**; each Notebook
   has the **Team/Personal** forking layers. The shared Workspace becomes genuinely
   shared (the convergence linchpin) and **visible**.
2. **Two admin tiers, both by reuse of existing/dormant infra** (no new role tables):
   - **Workspace admin (school level)** = `school_admins` + `is_school_admin()`
     (dormant today). Administers the whole Workspace: members/seats, all notebooks,
     workspace settings (calendar/holidays/week), create/archive notebooks, appoint
     notebook leads. The founding teacher is auto workspace-admin.
   - **Notebook lead (lead-teacher level)** = `grade_role 'lead'`/`'grade_admin'` on the
     TGA + `subject_team_memberships.can_edit_master`. Administers one Notebook: its
     Team/Master content, its membership (within the workspace's people), notebook
     settings, notebook-scoped invites.
3. **Multi-notebook ships now; multi-workspace stays deferred.** A teacher belongs to
   exactly **one Workspace** (scalar `teachers.school_id`). One account in *several*
   workspaces (R4) needs the scalar→junction RLS re-scope on live data — deferred,
   documented in §9. Admin works fine within one-workspace-per-teacher.
4. **Dynamic workflow.** Orchestrator owns the security-critical backend + every review
   gate + integration. Parallelizable surfaces are delegated to agents with
   **non-overlapping file ownership** and explicit contracts (§7).

## 1. Current-state facts (verified 2026-06-06)

- **Strategy A is live:** `teams.school_id` is `NOT NULL UNIQUE` → a team is **1:1 with
  a `schools` row = the Workspace** (`20260606120000_teams_invitations.sql:74`).
- **`teachers`** (`20260518102823_initial_schema.sql:217`): `id`=auth.uid,
  `school_id NOT NULL → schools`, `email`/`display_name` NOT NULL, `default_view`,
  `completion_privacy`, `default_grade_level_id`, timestamps. **One workspace per
  teacher (scalar `school_id`).**
- **`redeem_invite(p_token_hash text) returns table(redeem_status text, team_id uuid,
  grade_level_id uuid)`** (`20260606140000_invite_lifecycle.sql:350`): validates token
  hash + email-binding + seat cap (atomic, `FOR UPDATE`), then inserts
  `team_memberships` + **TGA** + per-role `subject_team_memberships` (master-edit only
  for lead/grade_admin), marks accepted, audits. **It does NOT touch
  `teachers.school_id`** — the workspace convergence is explicitly deferred to the auth
  path (its boundary comment, lines 54–67, 519–520). **This is the linchpin gap.**
- **Provisioning** `provision_individual_workspace` (`20260606130000`): seeds a full
  solo workspace (school + grade 'My Class' + 8 subjects + school_year + teachers row +
  owner team + membership). **Not invite-aware** (no pending-invite skip).
- **Authz helpers:** `can_read_grade(grade_level_id)` (TGA-or-school-admin) gates
  grade-scoped tables; `auth_teacher_school_id()` gates workspace-scoped tables
  (teammate list, school_years, frameworks, resources, reminders). **Because teammates
  will share `school_id`, both already isolate correctly once convergence lands.**
- **Auth paths:** Google GSI (`signInWithIdToken`) + Claude-bypass magic-link plumbing
  (`admin.generateLink` → `verifyOtp`) exist; **end-user `signInWithOtp` magic link is
  NOT wired**. Provisioning runs in `/auth/callback` + bypass via `ensure-teacher.ts`.
- **`school_admins` + `is_school_admin()`** exist and are **dormant** — the workspace-
  admin hook (decision #2).

**Net gap to close:** (a) workspace convergence so an invitee truly shares the
workspace; (b) the user-facing auth + invite + team surfaces; (c) multi-notebook
create/switch; (d) the two admin tiers (activate dormant infra + UI); (e) solo UX so
the workspace/team language only appears when there's a team.

## 2. Target architecture (additive to Strategy A)

```
Account (auth.users / teachers)  — one login, one Workspace (scalar school_id)
 └─ Workspace  = schools row   ── admin: school_admins (is_school_admin)
     │            settings: calendar / holidays / school_week / seats / members
     └─ Notebook = grade_levels row (+ subjects)   ── lead: TGA grade_role 'lead'
          ├─ Team      = master rows  (subjects.scope='team', STM.can_edit_master)
          └─ Personal  = personal copies / authored (scope='personal', owner-only)
   teams (1:1 schools, seat ledger ≤5)  ·  team_memberships  ·  invitations
```

- **Workspace shared** the instant `teachers.school_id` = the team's school for every
  member (convergence). The ~40 RLS policies are untouched.
- **Multiple Notebooks** = multiple `grade_levels` under one `schools` row (the original
  schema already permits this; `grade_levels` was intentionally not altered). Invitations
  already target a specific grade; per-notebook participation is per-grade TGA.
- **Admin** is a capability overlay, not a structural change: workspace-admin via
  `school_admins`, notebook-lead via `grade_role`.

## 3. Waves

### W-A — Backend: workspace convergence + admin scaffolding  *(orchestrator-owned; security-critical)*
- **Convergence migration** (`20260606150000_invite_convergence.sql`): fold convergence
  into `redeem_invite` (CREATE OR REPLACE, same signature) — after seat/validation and
  before the grants, **re-home the redeemer's `teachers.school_id` → the team's school**,
  with a **pristine guard**: re-home only when the redeemer's current workspace is the
  auto-seeded solo space with **no personal content** (no `personal_authored_lessons`,
  no `personal_core_lesson_event_copies`, no master edits they authored); otherwise
  return new status **`existing_workspace`** (defer R4, no data loss). On re-home, drop
  the redeemer's solo-space grants (their solo TGA/STM/owner-team/solo school rows that
  are now orphaned and pristine) so they don't see a phantom notebook. Re-runnable.
- **Provisioning invite-awareness** (`ensure-teacher.ts` + `/auth/callback`): when auth
  arrives with `next=/invite/[token]`, **do not auto-create a solo workspace**; route to
  redeem (which now provisions into the team). Fallback: if redeem fails
  (expired/seat-full), then create the solo workspace so the user is never stranded.
- **Admin scaffolding:** activate `is_school_admin()` for workspace-admin checks; seed
  the founding teacher as `school_admins`; confirm `grade_role` lead/admin gates. Add
  `audit_action` values for member/notebook/admin actions.
- **RLS:** keep the ~40 policies; add only invite/admin-specific policies needed for the
  new RPCs (TGA write stays RPC-only).

### W-B — Backend: multi-notebook + member/admin RPCs  *(orchestrator-owned)*
- `create_notebook(name)` (workspace-admin only) → new `grade_levels` + 8 subjects +
  links to the workspace's school_year; `rename_notebook`, `archive_notebook`
  (soft `is_active=false`).
- `set_member_role(teacher_id, grade_level_id, role)` / `remove_member` (workspace-admin
  or notebook-lead within scope) → adjust TGA + STM; removing a member frees a seat;
  Personal forks are retained (unreadable to them) not deleted.
- `grant_workspace_admin` / `revoke_workspace_admin` (workspace-admin only).
- All `SECURITY DEFINER`, capability-checked, audited, atomic.

### W-C — Frontend: auth surfaces  *(agent; owns `app/login/*`, `app/invite/[token]/*`, magic-link route)*
- Magic-link sign-in: a server route calling `signInWithOtp` (+ resend), confirm via the
  existing `/auth/callback`. Tabs on `/login`; reuse `safeRelativePath`, origin/CSRF
  guard from the gsi route. Keep Google.
- `/invite/[token]` accept screen: unauth → sign-in with `next` preserved; on auth,
  auto-redeem (hash the token, call `redeem_invite`); render every status
  (accepted / already_member / email_mismatch / expired / revoked / seat_full /
  invalid / **existing_workspace**) with a fallback "create your own space instead."

### W-D — Frontend: Team + admin settings  *(agent; owns `app/settings/team/*` + new components)*
- Member list (name, role, seat usage "3 / 5"), pending invites (revoke/resend), invite
  panel (link or email + role). Uses existing `create_invite`/`revoke_invite` +
  W-B role/remove RPCs; reads `team_memberships`/`invitations`.
- **Workspace-admin controls** (gated by `is_school_admin`): manage members across
  notebooks, appoint/revoke notebook leads, grant/revoke workspace-admin, workspace
  settings entry points. **Notebook-lead controls** (gated by grade_role): manage that
  notebook's membership + invites.
- Tooltips: consequential controls (invite, role, remove, admin grant) = **always-on**
  (`required:true`); others = dismissible W2-B3. No confirm dialogs.

### W-E — Frontend: workspace + notebook navigation  *(agent; owns the nav/switcher components + `lib`)*
- Surface the (previously hidden) **Workspace** and a **Notebook switcher** in the top
  chrome; "＋ New notebook" (workspace-admin) → `create_notebook`. Selected notebook
  drives the existing grade-scoped views. Solo users: switcher recedes to a single
  notebook; Workspace label stays quiet.

### W-F — Frontend: admin surfaces (foundation)  *(folds into W-D/W-E; orchestrator sequences)*
- Workspace admin panel (notebooks list: create/rename/archive, members, leads) and the
  notebook-lead admin view. Foundation-level now (essential actions); full admin console
  is later-phase.

### W-G — Solo UX, tooltips, responsive, tests  *(agent + orchestrator; cross-cutting, last)*
- Solo variants of the Personal/Team toggle + tooltip/settings copy (no "affects the
  whole team" when there is no team); flip to team language on 2nd member.
- Responsive at 360 / 768 / 1280 for every new surface; `vitest` for the convergence
  RPC (pristine re-home, existing_workspace, idempotency, seat cap) + RLS regression
  (A cannot read B; non-admin cannot call admin RPCs).

## 4. RLS / security must-holds

- **No cross-workspace leak:** after convergence, teammates share `school_id`; verify A
  (workspace 1) cannot read B (workspace 2) — regression test.
- **TGA/STM writes stay RPC-only**; no authenticated self-insert. Admin RPCs are the
  only privileged writers, each capability-checked + audited.
- **`can_edit_master` never defaults true on redeem** (already fixed; preserve).
- **Pristine guard on re-home is conservative** — never re-home (destroy/orphan) a
  workspace that has any personal content; defer to `existing_workspace`.
- **Admin checks server-side only** — UI gating is cosmetic; every admin RPC re-checks
  `is_school_admin()` / grade_role.

## 5. Risk register (top)

| Risk | Impact | Mitigation |
|---|---|---|
| Convergence re-home destroys/orphans real content | Critical | Conservative pristine guard; `existing_workspace` deferral; vitest covers it; review gate. |
| Cross-workspace leak | Critical | Shared school_id keeps the 40 policies correct; A-can't-read-B regression test. |
| Auth flow strands an invitee (no workspace) | High | Fallback solo provisioning if redeem fails; auto-redeem on the invite page. |
| Admin RPC privilege bug | High | Server-side capability re-check + audit on every admin RPC; independent-agent review. |
| Multi-agent file collisions | Med | Strict non-overlapping ownership (§7); orchestrator integrates + reviews. |
| Out-of-phase scope creep (multi-workspace, billing) | Med | Explicitly deferred (§9); admin built within one-workspace-per-teacher. |

## 9. Acceptance criteria

- A fresh invitee, via magic link or Google, accepting an invite **shares the workspace**
  (sees teammates, the shared year/calendar, the team's Master) and forks Personal copies
  — with **no throwaway solo space** and **no data loss** for any existing-content user
  (they get `existing_workspace`).
- A workspace can hold **multiple notebooks**; a workspace-admin can create/rename/archive
  them and appoint notebook-leads; a notebook-lead can manage that notebook's team.
- Solo teachers see **no team/workspace ceremony** until a 2nd member joins.
- All three responsive tiers pass; the gate passes; `lint`/`tsc`/`vitest`/`build` green.
