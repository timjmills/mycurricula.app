   until cutover completes (Redesign 1/2 stand down briefly).
2. **Apply the two migrations** (additive, v1-safe — prod keeps working):
   `npx supabase db push --linked` from a checkout of the v2 tip (or apply
   the two files via the SQL editor). Verify with `migration list --linked`.
3. **Merge:** open + merge the PR `claude/v2-wave3-chrome` → `master`
   (merge commit, repo convention).
4. **Deploy watch:** the push triggers "Deploy to Cloudflare"; the build runs
   `check-v2-flag --build` with `NEXT_PUBLIC_V2='1'` (workflow default).
   Watch it green.
5. **Prod verify (same session):** claude-login → /home /daily /weekly /year
   /planner /catch-up /post render v2; console sweep; theme flip; the
   `mc-theme-axes` SSR cookie present. Verify theme-sync now persists
   (teacher_preferences row gains the v2 axes on an appearance change).
6. **Announce:** log the cutover in `agent_shared_log.md`; unfreeze.

## Rollback (if prod breaks)

- **UI rollback (minutes):** set repo Variable `NEXT_PUBLIC_V2=0` → re-run the
  deploy workflow. Ships the v1 UI from the same tree (router+chrome gates,
  tested by the W13 regression). Migrations stay (additive/nullable — v1 code
  never reads the new columns).
- **Full revert (worst case):** revert the merge commit on master → auto-
  deploy rebuilds prior prod. Migrations again safe to leave.
