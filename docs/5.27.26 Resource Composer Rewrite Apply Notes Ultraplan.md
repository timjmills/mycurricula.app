2. Slice 2 (`5.27.26 Resource Embed Render Swaps (Ultraplan).patch`) — not strictly required for this patch to *apply*, but needed for the visible demo (without Slice 2 the rendered composer chip's `<ResourceEmbed>` preview shows but the lesson surfaces still draw synthetic glyphs).

Apply order: **Slice 1 → Slice 2 → Slice 3**, committing each in between.

## What it does

Wires the Daily-view composer (`ResourceComposer.tsx`) against the new resource embed primitives, plus extends `planner-store.tsx`'s `addSectionResource` mutator and adds a new `editSectionResource` mutator for display-mode toggles.

After this patch:

- **Paste a YouTube / Vimeo / Google / Drive / website URL into the composer** → `parseResourceUrl` runs immediately. The chip shows a live `<ResourceEmbed>` preview inline. Add commits the full `LessonResource` shape to the planner store.
- **Pick / drag / paste a file** → in-session `blob:` URL is minted via `URL.createObjectURL`. Renders in the lesson immediately on Add. Survives the session, not a reload (real R2 persistence lands when the mocks→DB bridge ships).
- **Link display mode toggle** — every captured website link chip carries a 3-way segmented control (literal · hyperlink · thumbnail) that the teacher flips per-link. The choice flows through to the renderer via `LessonResource.displayMode`.
- **Limits banner** — live count above the captured-items strip: `"N / 10 files · N / 10 images · N link(s)"`. The 11th file or image is silently dropped at capture time so the teacher can't even queue an over-limit item.
- **Blob URL cleanup** — every `URL.createObjectURL` minted in a session is revoked on dialog close so we don't leak.

## What's NOT in this patch

- **Real R2 upload from the composer.** The infrastructure ships in Slice 1 (`lib/r2.ts`, `/api/resources/upload`, `/api/resources`), but wiring the composer's file-pick path to it requires real UUIDs for `owner_event_id` — today's planner store works with string IDs (`"m-11-1"`). When the mocks→DB bridge lands as part of the broader Phase 1B work, swap the in-session `URL.createObjectURL` path in the composer for a `fetch("/api/resources/upload")` → PUT to R2 → `fetch("/api/resources")` chain. The API contract is already documented in `docs/5.24.26 claude-access.md`.
- **PATCH-the-row-on-display-mode-change**. Today the display-mode toggle writes through `editSectionResource` to in-memory state only. When the backend wave lands, that mutation should fire `PATCH /api/resources/[id]` so the change persists. Easy follow-up — the route exists.
