# Performance recon — v2 branch, pre-cutover

> **Snapshot disclaimer (CLAUDE.md §8):** dated audit snapshot, 2026-07-16.
> Static analysis of `origin/claude/v2-wave3-chrome` + measured route tables
> from isolated production builds (flag-ON and flag-OFF). Verify against
> current code before acting. Report-only.

## Measured route weights (first-load JS)

/home **792 kB** (default landing) · /weekly 603 · /year 602 · /daily 548 ·
/teach 504 · /catch-up 332 · shared-by-all 103 (= React/Next runtime chunks —
not actionable). Flag-ON and flag-OFF builds are near-identical per route.

## Findings

1. **`lib/home/insights.hero.json` = 426 kB statically imported client-side**
   (`lib/home/insights.ts:11`; consumers: `components/chrome/ChromeQuote.tsx`
   + the v1 home tree). Parsed composition: 320 records, **~280 kB (66%) is
   `expand` prose used ONLY in click-to-open expansions**
   (`ChromeQuote.tsx:275`, `RollingInsight.tsx:127`); quotes themselves are
   only ~49 kB. Nothing at first paint needs `expand`.
2. **Dual-branch shipping on gated routes — UNCONFIRMED but risk factors
   confirmed:** near-identical ON/OFF sizes are suggestive (not proof); the
   classic tree-shaking-defeat pattern IS present — **no `sideEffects` field in
   package.json + v1 fallbacks imported via barrels** (`HomeV1` via
   `@/components/home` which re-exports the whole home tree). Applies to all 5
   gated routes. Decisive experiment: build as-is vs with the v1 import
   removed; diff the route's first-load.
   **Fix if confirmed:** NOT `next/dynamic` per branch (forces client-render,
   hurts FCP/streaming AND rollback fidelity). Instead: `sideEffects` hint in
   package.json (**use `["*.css"]`-style allowlist, not bare `false` — CSS
   imports are real side effects**) + deep-path imports for the v1 fallbacks so
   Terser constant-folds `V2 ? A : B` and DCEs the dead branch, keeping SSR.
3. **Standards catalog on /daily:** `lib/standards/frameworks-catalog.json`
   (159 kB) static-imported by `lib/standards/catalog.ts:14`; only consumer is
   `StandardsPicker` → `PlanningTabs` → (v1) `LessonDetail`/`DailyViewV1`.
   Server endpoints already exist (`/api/standards/facets|frameworks|search`).
   **Interaction note (orchestrator):** under flag-ON this whole chain is the
   DEAD branch — fixing finding 2 may drop the catalog from /daily for free;
   the fetch-instead-of-import change then only matters for the flag-OFF/v1
   artifact.
4. **Mock fixtures ship to prod:** `lib/planner-store.tsx` statically imports
   `lib/mock` (fallback catalog at module load; `lib/mock/lessons.ts` alone
   41 kB) — every planner route carries it even with Supabase ON. 79 files
   import `lib/mock`; deeply woven → awareness, not a quick win.
5. **rich-text-editor (76 kB, hand-rolled)** statically imported into ~10
   daily/lesson components but only used in EDIT mode → `next/dynamic`
   candidate on /daily.
6. **Hero photos:** CSS `background-image` webp files (65–132 kB × 15+ in
   `public/classroom/`) — not in JS bundles; a `next/image` treatment would
   trim network, not first-load JS. Minor.

## Cleared (no win — don't chase)

Shared 103 kB = framework runtime · no @react-pdf/xlsx/chart/date libs bundled ·
`pdfjs-dist` already dynamic · @dnd-kit + DOMPurify are legit core/static.

## Ranked wins (est. kB × ease)

1. Split `expand` out of insights.hero.json → fetch on open. **~280 kB off
   /home.** Ease HIGH, risk LOW. Best win.
2. Confirm + fix dual-branch DCE (finding 2). Potentially largest aggregate win
   (5 routes); confirm first. Ease MED, risk LOW with SSR-preserving approach.
3. Standards catalog → API fetch (may be subsumed by #2 for flag-ON; still
   trims the v1/rollback artifact). ~159 kB. Ease MED.
4. `next/dynamic` ChromeQuote in ChromeShell (cross-route magnitude UNCERTAIN —
   /catch-up at 332 kB suggests some splitting already happens; verify).
5. `next/dynamic` the rich-text editor (~76 kB off /daily first paint).
