## 0. Decisions locked with the product owner

1. **Handoff is the new source of truth.** Refactor Teach to match the 5.31
   handoff; salvage the repository/tag/cap foundation already built where it
   still fits; replace UI that diverges.
2. **Free-form canvas** replaces the fixed CSS grid (drag anywhere, corner-
   resize, per-widget restyle).
3. **Widget set = superset.** Build the ~25 named pedagogical widgets AND keep
   the existing generic ones, **restyled to the new pastel visual system**;
   **delete exact duplicates** (the survivor is the named widget).
4. **Repeat = ONE board scheduled into many contexts** (weekday/time/daily/
   weekly/subject/slot). It is the same board — editing it changes every
   occurrence. Repeat is NOT independent copies. **Repeat creates REAL links
   now** (decision 2026-05-31): a repeat references actual schedule-slot /
   lesson / day / week / subject entities (not cosmetic labels), so a repeated
   board genuinely surfaces in those contexts via the planner's own selectors.
   The real-entity shapes come from the schedule/lesson-model survey (Wave A.0).
5. **A board can have multiple PAGES.** Board → Page[] → Widget[]. The `‹ 9 ›`
   fullscreen page nav is real.
6. **Nav order** (top bar): Daily · Weekly · Yearly · Curriculum · **Teach**
   (Teach last). Labels unchanged. Already true on the working branch.
7. **Build with agent teams**, mock-data only (Phase-1A style) through the
   `TeachDataSource` seam; Supabase deferred. Ship as one tested package.

## 2. Data model (additive, mock-first)

### 2.1 Board → Pages → Widgets

```
Board
  id, title
  tags: BoardTag[]                 (subject/lesson/phase/day/week/slot/label/type)
  repeat: RepeatSchedule | null    (one board, many scheduled contexts)
  boardTheme: ThemeOverride        (global appearance, overridable per widget)
  pages: BoardPage[]               (≥1; the page nav cycles these)
  ownerId | scope | libraryVisibility | publishedBy | sourceBoardId   (already built)
  background, whiteboard, ephemeral, createdAt, updatedAt              (already built)

BoardPage
  id, order
  widgets: CanvasWidget[]

CanvasWidget            (free-form placement — replaces WidgetGridPosition)
  id, type, x, y, w     (absolute px; height flows from content; clamp w 230–640)
  ov: ThemeOverride     (per-widget appearance override; {} = inherit board theme)
  data: Record<string,unknown>   (structure only — NO student names, §11.4)
  pinned, persistence, gradeLevelId
```

`RepeatSchedule` — **real links, not labels** (decision 2026-05-31). Each rule
carries the actual entity id(s) from the planner model (filled in by the Wave
A.0 schedule/lesson survey), plus a derived display label:
```ts
type RepeatKind = "weekday"|"time"|"daily"|"weekly"|"subject"|"slot"|"lesson";
interface RepeatRule {
  kind: RepeatKind;
  // REAL references — resolved against the planner's own entities:
  weekdays?: number[];   // 0-based indices into the CONFIGURED school week
  slotId?: string;       // a real schedule-slot / period id
  lessonId?: string;     // a real master lesson id
  subjectId?: SubjectId; // one of the 8 locked subjects
  week?: number;         // a real week number/id
  label: string;         // derived for display (e.g. "Mon/Wed/Fri", "Daily")
}
type RepeatSchedule = RepeatRule[] | null;  // a board may repeat by >1 rule
```
A board's repeat is resolved through the planner's existing selectors (e.g.
"lesson for day D / week W / subject S", "slot at time T") so the link is live,
not a string. The board appears wherever its rules match — and because it is one
board, editing it changes every occurrence. **Wave A.0 (schedule/lesson-model
survey) supplies the exact id fields these reference before the model is frozen.**

`ThemeOverride` (the appearance contract):
```ts
interface ThemeOverride {
  bg?: BgKey;       // yellow|green|pink|purple|orange|blue|slate|cloud|dark
  accent?: AccentKey; // blue|green|purple|orange|pink|yellow|slate|ink
  text?: "ink"|"slate"|"white";
  size?: number;    // 0.8–1.4
  radius?: number;  // 6–30
  font?: "jakarta"|"rounded"|"serif"|"hand"|"mono";
}
```
Effective theme = `merge(widgetDefault, boardTheme, widgetOverride)` with the
dark-bg→white-text auto rule (per handoff `effective()`/`clean()`).

- `WidgetGridPosition {col,row,colSpan,rowSpan}` → `{x,y,w}` (free-form). Provide

  subject, or tag tokens (verified by Agent 2).
- The **themeable-widget contract**: `.tw` reads `--w-grad/-soft/-chip/-line/
  -card/-accent/-ink/-radius/-font/-scale`; internal sizes use `em` so
  `--w-scale` rescales the whole widget.

### 4.3 Interactivity (client-only, structure-only state)

Interactive: Timer, Name Picker, Scoreboard, Dice, Countdown, Work Sound, Poll,
Understanding Check, Help Queue, Participation Tracker, Brain Break, Exit Ticket,
Calm Corner, Mini Whiteboard, Lesson Flow (reorder), Student Groups (slots).
All others display-only. State persists via the existing `use-widget-state`
seam (localStorage now). **No student names in any widget `data`/`state`.**

### 4.4 PRIVACY (hard rule, §11.4)

Widgets that show people — Student Groups, Name Picker, Participation Tracker,
Help Queue, Student Jobs, Work Completed, Teacher Table — render **initials-on-
tint avatars only**. Names live solely in `useTeachGroups()` (local, USER-scoped).
The `Avatar` component hashes name→hue but renders only the initial. Nothing
name-bearing reaches board/widget config, state, fixtures, or the repo.

## 8. Acceptance criteria (no gaps/bugs/omissions bar)

- All ~40 widgets render against `--w-*`; each has a default theme; per-widget
  + board theme overrides compose via `effective()`.
- Free-form editor: place/drag/resize/duplicate/delete; pages add/navigate;
  appearance persists; resource drop works.
- Fullscreen: background presets, favorites add, library add, all 6 markup
  tools + undo/redo/clear, side-flip, page nav, Esc/exit.
- Library: 50-cap enforced; tags + Repeat shown; Duplicate (independent) vs
  Repeat (same board, many contexts) behave per §0.4; Share publishes a copy;
  pull counts to cap.
- **Privacy:** grep proves no student names in fixtures/config/state.
- **Tokens-only:** no hex / no px font-size in components (grep gate).
- **Responsive:** no doc-level horizontal scroll at 360/768/1280; touch targets
  ≥44px; sticky chrome ≤30% vh on phone.
- **A11y:** present/fullscreen focus-trap + Esc; tab order documented; reduced-
  motion respected; widget contrast AA.
- `tsc --noEmit` 0, `next lint` clean, `next build` green.
