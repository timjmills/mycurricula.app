## 2. The product goal (explicit + implicit)

- **Explicit:** one curriculum operating system per teaching team — git-style forking
  (one Team plan, automatic Personal copies), friction only where it matters.
- **Implicit (from doc trajectory):** teacher-first SaaS — solo signup → 5-seat team
  invites → dormant school/org layer; Teach mode + the notecard/resource system are
  the second moat.
- **Senior-UX thesis of this review:** the biggest friction risk is no longer missing
  features — it's **promises the UI doesn't keep** (dead-end buttons, placeholder
  panes, decorative chrome). Teachers forgive missing features; they don't forgive
  broken trust.
