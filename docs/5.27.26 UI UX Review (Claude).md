# 5.27.26 UI/UX Review — New-Teacher First-Visit Experience

> **Date:** 2026-05-27
> **Scope:** Phase 1A beta-ready prototype at `mycurricula.app`, branch `schedule-and-auth-5.24`
> **Author:** Claude (commissioned audit)
> **Lens:** how a new teacher who has never used the website might interact

---

## Context — why this audit, why now

Phase 1A is shipping. The Grade 5 beta with Tim's team opens late August. The investment to date is heavy on **explanation at the point of contact** — 686 `<Tooltip>` callsites across the codebase, onboarding voice on every named panel, the MasterBanner pulse, the ArchiveToast undo, the "= N weeks" live readout, the celebratory Catch-up empty state. CLAUDE.md §4 (the onboarding tooltip mandate) is well-internalized component by component.

The next gap is **structural, not local**. A first-time teacher's friction is concentrated in four areas:

1. **Discovery** — features they don't already know exist (rails are drag-arrangeable; the chameleon header is meaningful; Catch-up is a real concept, not a label).
2. **Concept teaching** — three load-bearing ideas (Personal vs. Master, Catch-up, the chameleon header) carry team-level consequences but only reveal stakes on hover or *after* the action.
3. **Consequence feedback** — toggling team settings is silent; the teacher can't tell whether a switch was personal or team-wide.
4. **Scope clarity** — Settings tabs split team-shared vs. user-local, but the split is buried in card body text.

The cost of getting this wrong is asymmetric. A first-time teacher who flips Master, edits something, and panics is worse than one who never finds a feature. This review extends the existing tooltip investment into **discovery + concept-teaching + consequence-feedback**.

## Section 2 — Top 10 first-visit friction points

Concrete, ranked, with file:line cites.

1. **Personal/Master toggle — full consequence only in tooltip** — `components/shell/top-bar.tsx:473-497`. Labels say "Personal | Master"; "changes affect the whole team" lives only in the Tooltip + title=. The red MasterBanner is *post-edit* feedback, not a *pre-edit* gate. A teacher who clicks Master without reading the tooltip creates shared edits with no undo.

2. **"Catch-up" terminology unexplained at entry points** — `components/catchup/CatchupScreen.tsx` has a strong subtitle, but the flame badge (`components/shell/catchup-flame-button.tsx`) and the in-grid Catch-up bar don't carry the same concept-teaching. A teacher sees "Catch-up" without knowing it means "uncovered lessons that need a make-up plan."

3. **Weekly-view lesson reorder is local-only** — `components/daily/DailyView.tsx:54-62`. Drag affordance is clear; the data store comment says "purely a personal viewing preference…never written to the planner store." Newcomers assume reorder syncs back to the team.

4. **Daily detail splitter hidden at ≤720px** — `components/daily/DailyView.module.css:997-1018`. Below 720px the splitter vanishes. Lesson list shows by default; clicking swaps to full-width detail with a "← Back" button. The back button is **inside** the detail header, not sticky — scrolls out of view.

5. **Left filter panel starts CLOSED on /subject, OPEN elsewhere** — `components/shell/left-filter-panel.tsx:15-18`. The one-time exception breaks the pattern. (⚠ verify — the Plan agent flagged that current code may not match this finding.)

6. **Chameleon header tint is a silent indicator** — `components/year/QuarterMonthWeekHeader.tsx:54-62`. The header tint changes as you scroll through Year lanes; no aria-label, no tooltip explains it. A new teacher might think it's a highlighting bug.

7. **Unit Health stat labels** — `components/subject/UnitHealthCard.tsx:148-194`. The Explore agent mentioned abstract glyphs; the Plan agent's verification found that the current card uses a progress bar + labeled stat tiles (STANDARDS / SKIPPED / WHEN), not glyphs. **Real gap:** the STANDARDS stat has no tooltip explaining "fraction of distinct standards covered by completed lessons."

8. **SOON-tab fallback at phone width** — `components/shell/top-bar.tsx:283-335`. At ≤480px Yearly + Curriculum tabs hide. Need to verify the More menu (`top-bar-more-menu.tsx`) renders them with the SOON badge so phone users can still discover them.

9. **Schedule mode = per-view pill, not top-bar Grid/List** — `components/shell/top-bar.tsx:501-528`. Naming collision: teachers expect the top-bar Grid/List toggle to show a timetable. The Schedule mode pill lives in each view's own chrome.

10. **Information density at 1280px** — `components/daily/DailyView.module.css:1560-1600`. Four-track layout fits at 1280; below 1280 the schedule rail collapses, below 720 the splitter hides, below 480 the week-strip pills scroll internally. Affordances disappear in sequence; no breadcrumb across the cascade.

## Section 3 — Recommended fixes (by theme)

### A. Concept teaching

- **A1. Personal/Master toggle — pre-edit tutorial popover** on first flip to Master. Anchored under the toggle, dismissible with "Got it", persisted via `mycurricula:user:master-mode-introduced`. The MasterBanner heads-up phase already runs *post*-flip; this popover educates *before* the first edit.
- **A2. Catch-up — entry-point tooltips name the concept.** Flame badge + in-grid Catch-up bar lead with "Catch-up = lessons that fell behind and need a make-up plan."
- **A3. Chameleon header — name the signal.** Wrap header in `<Tooltip content="Header color tracks the subject lane you're viewing — it shifts as you scroll between subjects" side="bottom">`. Enrich `aria-label` to include active subject name when `subjectId` is set.
- **A4. Unit Health card — subtitle + STANDARDS stat tooltip.** Add 1-line subtitle below unit name; wrap STANDARDS label in Tooltip.
- **A5. Shoutbox tabs — first-time clarity.** Day Shoutbox tooltips on "Team chat" pill explain surface + jump-to-lesson behavior.

### B. Discovery affordances

- **B1. Rail drag-affordance.** Add `cursor: grab` / `cursor: grabbing` in `GlobalRail.module.css` + `RightIconRail.module.css`. Add a once-per-teacher 2-second pulse on the first rail icon (gated by `mycurricula:user:rails-drag-introduced`).
- **B2. "?" help button — top-bar entry point.** New control in the right cluster next to Profile. Click opens a route-aware `<HelpDrawer>` whose copy registry lives in `lib/help-copy.ts`. Every drawer carries a "Replay the onboarding tour" CTA.
- **B3. Product-tour re-entry.** Surface "Replay the onboarding tour" in the HelpDrawer AND on `/settings` landing as a new card. Both route to `/onboarding`; the wizard is resumable via `lib/onboarding-state.tsx`.

### C. Consequence feedback

For every team-scoped setting toggle, fire a `<ConsequenceToast>` — new thin wrapper around the ArchiveToast visual pattern.

| ID | Setting | Toast copy |
|---|---|---|
| C1 | Catch-up on/off | "Catch-up cues turned OFF for everyone on your team." |
| C2 | Curriculum label edit | "Curriculum name updated — every teacher on your team sees the new label." |
| C3 | Holiday add/remove | "Added [name] — every teacher's planner now skips this day." (5s undo) |
| C4 | Academic year save | "School year set to N weeks — applies to the whole team." |
| C5 | School-week change | "School week changed. Existing lessons reindexed to match." |

Build pattern: `lib/consequence-toast.tsx` exposing `useConsequenceToast()` returning `{ show(msg, opts) }`. Mounted once at planner shell + settings layout.

### D. Scope clarity

- **D1. Settings sidebar — TEAM / YOU chips.** `app/settings/layout.tsx:40-49` (extend TABS shape with `scope: "team" | "user"`). Sidebar chip + tooltip explaining the distinction. Curriculum → TEAM; Appearance, Catch-up, Lesson templates → YOU.
- **D2. SettingsCard scope tags.** `components/appearance/settings-card.tsx` grows optional `scope` prop. Migrate Catch-up + Curriculum + Holiday cards.
- **D3. Personal/Master toggle — scope chip in Tooltip content.** Tooltip primitive accepts ReactNode; render a tiny scope chip above the explanation copy.

### E. Empty states + intro subtitles

Reuse `components/ui/EmptyState.tsx` + existing PageHeader pattern.

- **E1. Daily — empty lesson list.** "No lessons for this day yet — Add a lesson with the + button above, or check a different day."
- **E2. Daily — center detail empty.** "Pick a lesson on the left — Select any lesson row to see its sections, resources, and notes."
- **E3. Intro subtitles per main view** (gated `mycurricula:user:<view>-intro-seen`):
  - `/daily`: "Today's lessons. The center panel follows whichever row you select."
  - `/weekly`: "The team's master grid — every subject, the whole school week."
  - `/year`: "Your subjects' road across the school year. The header tints to whichever subject you're scrolling through."
  - `/subject`: "One subject's units, standards covered, and your team's don't-miss callouts."

### F. Responsive collapse cliffs

- **F1.** Verify the More menu's NavigationSection renders SOON-flagged tabs with the badge inline so phone users can discover Yearly + Curriculum.
- **F2.** Sticky back affordance in narrow Daily detail. `position: sticky; top: 0; z-index: 5; background: var(--paper);` on the back-button row.
- **F3.** Phone-width responsive cliff audit (Wave 3). Walk 360/768/1280 across every view; document every control that disappears with no fallback.

### G. Visual / formatting polish

- **G1.** Verify SOON badge background contrast.
- **G2.** Master banner persistent strip at 360px — verify text doesn't wrap/overflow.
- **G3.** Document the responsive cascade for /daily resource grid + to-do panel + shoutbox panel.
