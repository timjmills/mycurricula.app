# Resource embed render swaps — apply instructions

**Patch:** `5.27.26 Resource Embed Render Swaps (Ultraplan).patch` (4 files, 321 lines)
**Depends on:** Slice 1 — `5.27.26 Resource Embed Slice (Ultraplan).patch` must be applied first (this patch imports from `@/components/resources`, which Slice 1 creates).

## What it does

Adds real-embed render paths to the three resource render sites + sprinkles six real URLs into the mock fixtures so the `/daily` demo works without any backend or composer changes.

After this patch:

- Open `/daily`, navigate to a lesson with a seeded URL — the synthetic icon is replaced by a real YouTube player, Vimeo, Google Slides, Google Doc, Drive PDF preview, or public image (one per fixture).
- Open `/subject/<id>` — the all-resources sort table renders real thumbnails inline.
- Lessons without seeded URLs render exactly as today (the conditional is `resource.url ? <ResourceEmbed> : <synthetic>`).

## What's NOT in this patch

The composer still produces synthetic `{ type, label }` items — pasting a URL there won't yet flow through the parser to produce a real embed (the link path stays as the legacy single-label form). That's the next slice (E1 + E2): planner-store action extension + ResourceComposer rewrite.

What works WITHOUT the composer rewrite:

- All six seeded fixtures render their real embeds at `/daily` and `/subject/*`.
- Anything in the mock that doesn't have a URL keeps its synthetic glyph (no regression).

What does NOT work yet:

- Pasting a YouTube URL in the composer → still saves as a synthetic link, not a real embed.
- File uploads through the composer.
- The 3-way display-mode toggle on link cards.

Those land in the next patch.
